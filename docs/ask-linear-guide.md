## 什么是 Ask Linear

Ask Linear 是 Linear 提供的 AI 问答功能，让你可以用日常语言与工作区中的数据进行交互。你无需记住复杂的筛选条件或快捷键，只需输入一句话，即可获得相关 Issue、项目或文档的即时回答。

**核心能力包括：**

- 🔍 **自然语言检索**：用描述代替筛选条件
- 📋 **内容汇总**：自动整理多个 Issue 的关键信息
- 📊 **状态分析**：快速了解某个项目、迭代或团队的当前进展
- ✏️ **辅助创建**：根据描述起草 Issue 标题与内容
- 🔗 **关联发现**：找到与当前工作相关的其他 Issue 或文档

---

## 如何打开 Ask Linear

1. 在任意页面按下快捷键 `/` 或点击侧边栏底部的 **Ask Linear** 入口
2. 在弹出的输入框中输入你的问题
3. 按回车键提交，稍等片刻即可看到回答

> 💡 也可以在 Issue 详情页内直接调用 Ask Linear，回答将自动结合当前 Issue 的上下文。

## 高频使用场景

### 1. 查询 Issue 信息

快速找到某个 Issue 或了解其详情，无需手动搜索。

**示例提问：**

```
上周 @张三 负责的所有 Issue 有哪些？
```
```
CITRO-749 这个 Issue 目前的状态是什么？
```
```
有没有关于"登录页面"的 bug？
```

**适用场景：** 日常站会前快速回顾待办、帮助新成员快速了解历史任务。

---

### 2. 搜索与过滤

替代复杂的手动筛选，通过描述找到你需要的 Issue 列表。

**示例提问：**

```
本迭代中还没完成的前端任务有哪些？
```
```
优先级为「紧急」且还未指派负责人的 Issue 有几个？
```
```
过去两周内被关闭的 Issue 里，哪些属于 Bug 类型？
```

**适用场景：** Sprint 复盘、排期规划、数据统计。

---

### 3. 总结与摘要

将多个 Issue 或一段时间内的工作整理成简洁摘要，方便对外汇报或留存记录。

**示例提问：**

```
帮我总结一下「用户认证」模块本周完成了哪些工作。
```
```
把 CITRO-700 到 CITRO-720 这批 Issue 做一个简要概览。
```
```
这个 Project 目前的整体进展如何，有没有阻塞项？
```

**适用场景：** 周报撰写、项目汇报、管理层同步。

---

### 4. 进度与状态分析

了解某个团队、项目或迭代的整体健康状况。

**示例提问：**

```
当前 Sprint 的完成率是多少？
```
```
「移动端」团队这个月的吞吐量和上个月相比如何？
```
```
哪些 Issue 已经超过预估时间但还未完成？
```

**适用场景：** 迭代中期检查、团队效能分析、风险识别。

---

### 5. 创建与更新 Issue

利用 Ask Linear 辅助起草 Issue，或对现有 Issue 进行批量描述性更新。

**示例提问：**

```
帮我起草一个关于「首页加载速度优化」的 Issue，优先级设为高。
```
```
参考 CITRO-600 的格式，为「黑暗模式支持」功能创建一个新 Issue。
```

**适用场景：** 快速拟写任务描述、保持 Issue 格式统一。

---

### 6. 关联内容发现

找到与当前工作相关的 Issue、文档或历史讨论，避免重复工作。

**示例提问：**

```
有没有和「支付流程」相关的已有 Issue 或讨论？
```
```
我正在处理 CITRO-749，有没有类似的 Issue 曾经被解决过？
```
```
「数据导出」功能有没有相关的设计文档或需求背景？
```

**适用场景：** 开始新任务前的背景调研、防止重复造轮子。

---

### 7. 团队协作辅助

帮助团队成员了解彼此的工作，减少沟通成本。

**示例提问：**

```
@李四 今天在处理哪些任务？
```
```
评审会之前，帮我列出所有需要讨论的 Issue。
```
```
哪些 Issue 被阻塞超过 3 天了，负责人是谁？
```

**适用场景：** 团队日会、资源调配、阻塞问题跟进。

## 提问技巧

| 技巧 | 说明 | 示例 |
|------|------|------|
| **指定时间范围** | 限定查询的时间窗口，结果更精准 | "本周"、"过去 14 天"、"上个迭代" |
| **指定负责人** | 使用 @ 提及团队成员 | "@王五 负责的 Issue" |
| **指定项目或标签** | 帮助缩小检索范围 | "属于「后端」团队的"、"标记为 Bug 的" |
| **使用 Issue 编号** | 直接引用具体 Issue | "CITRO-749 的详情" |
| **要求格式化输出** | 让回答更易于复制和使用 | "用列表形式展示"、"给我一个摘要" |
| **提供上下文** | 补充背景信息，提升回答质量 | "我正在做 XX 功能，想知道..." |

---

## 注意事项

- Ask Linear 的回答基于你所在工作区内的数据，**不会访问工作区外的信息**。
- 如果回答不够准确，可以尝试**换一种表达方式**或补充更多上下文再次提问。
- Ask Linear 目前**不支持直接修改数据**，创建或更新 Issue 的操作仍需在界面上确认。
- 涉及敏感数据时，请注意 Ask Linear 的访问权限与团队的数据安全规范保持一致。
